## 🔧 Technical Decisions

### Framework Choices

| Decision | Rationale |
|----------|-----------|
| **Vite over Webpack** | Faster HMR, simpler config, better DX |
| **Tailwind v4** | Latest features, CSS-first approach, built-in @tailwindcss/postcss |
| **Zustand over Redux** | Less boilerplate, better TypeScript, smaller bundle |
| **Axios over fetch** | Interceptors, request/response transformation, wider compatibility |
| **React Router v7** | Latest features, excellent TypeScript support |

### Architecture Patterns

1. **API Layer Pattern**
   - Single axios instance with interceptors
   - Token management centralized
   - Auto-refresh on 401

2. **State Management**
   - Zustand for global state (auth, player)
   - TanStack Query for server state (future)
   - Local component state for UI

3. **Component Organization**
   - Feature-based folders
   - Shared UI components
   - Layout components separate

4. **Routing Strategy**
   - Protected routes wrapper
   - Nested routes with layout
   - Role-based access (future)

## 🐛 Issues Encountered & Resolved

### Issue 1: Tailwind CSS v4 Configuration
**Problem:** Build failed with PostCSS error
```
Error: It looks like you're trying to use `tailwindcss` directly as a PostCSS plugin
```

**Root Cause:** Tailwind v4 changed PostCSS plugin structure

**Solution:**
```bash
npm install -D @tailwindcss/postcss
```

Updated `postcss.config.js`:
```javascript
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

### Issue 2: TypeScript Type Import Errors
**Problem:** Build failed with type import errors
```
error TS1484: 'User' is a type and must be imported using a type-only import
```

**Root Cause:** TypeScript `verbatimModuleSyntax` setting

**Solution:** Changed all type imports:
```typescript
// Before
import { User, Playlist } from '../types';

// After
import type { User, Playlist } from '../types';
```

### Issue 3: Environment Variables Not Working
**Problem:** `import.meta.env.VITE_API_BASE_URL` undefined

**Root Cause:**
1. Missing `VITE_` prefix
2. No `.env` file created
3. Missing TypeScript definitions

**Solution:**
1. Created `.env` file with `VITE_API_BASE_URL`
2. Created `src/vite-env.d.ts`
3. Updated `src/api/axios.ts` to use env var
